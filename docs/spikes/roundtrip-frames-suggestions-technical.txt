0:"The `ComboBox` and `Picker` components in React Spectrum (v3) both allow users to select a single option from a collapsible list. However, they differ in their primary interaction model:\n\n**ComboBox:**\n*   **Interaction:** The `ComboBox` is primarily driven by an input field. Users can type to filter the options, and the dropdown appears as they type. It's suitable when you want to provide an input for users to filter or type in a new value.\n*   **Use cases:**\n    *   Selecting an item from a long list where filtering is essential.\n    *   Allowing users to input custom values that are not in the predefined list (using `allowsCustomValue`).\n    *   Autocompletion scenarios.\n*   **Import:** `@react-spectrum/s2/ComboBox`\n*   **Key Props:** `allowsCustomValue`, `inputValue`, `onInputChange`, `menuTrigger` (to control when the menu opens, e.g., `input`, `focus`, `manual`).\n\n**Picker:**\n*   **Interaction:** The `Picker` presents a button-like element that, when clicked, opens a list of options. It does not have an input field for typing/filtering by default, making it more suitable for shorter, well-defined lists.\n*   **Use cases:**\n    *   Selecting from a short, predefined list of options.\n    *   When visual selection is preferred over text input for filtering.\n    *   When you want to ensure the user selects from the exact options provided.\n*   **Import:** `@react-spectrum/s2/Picker`\n*   **Key Props:** `selectionMode` (can be `single` or `multiple`), `defaultValue`, `value`, `onChange`, `renderValue` (for custom display of selected items).\n\n**Key Differences Summarized:**\n\n| Feature          | ComboBox                                      | Picker                                       |\n| :--------------- | :-------------------------------------------- | :------------------------------------------- |\n| **Primary Input**| Text input field                              | Button to open a dropdown                    |\n| **Filtering**    | Built-in filtering via typing                 | No built-in filtering (can be added with custom logic or if the list is short) |\n| **Custom Values**| Supports `allowsCustomValue`                  | Does not support custom values directly      |\n| **Use Case**     | Long lists, filtering, autocompletion, custom input | Shorter lists, direct selection              |\n| **Import Path**  | `@react-spectrum/s2/ComboBox`                 | `@react-spectrum/s2/Picker`                  |\n\n**When to Use Which:**\n\n*   **Use `ComboBox`** when you have a large number of options, and users will benefit from typing to filter the list, or when you need to allow users to enter custom values.\n*   **Use `Picker`** when you have a smaller, defined list of options and the primary interaction is to simply select one or more from the list without the need for text input filtering.\n\nYou can find more details and examples in the React Spectrum documentation:\n*
